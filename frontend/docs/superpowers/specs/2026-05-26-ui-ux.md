> **Documentos relacionados**: [PRD](./2026-05-26-prd.md) · [TRD](./2026-05-26-trd.md) · [Flujos](./2026-05-26-flujos.md) · [Schema](./2026-05-26-backend-schema.md) · [Plan](./2026-05-26-plan-implementacion.md)

## Principios de Diseño

| Principio | Descripción |
|-----------|-------------|
| **Claridad** | La interfaz debe comunicar el valor en segundos. Nada de jerga técnica para el usuario final. |
| **Confianza** | Diseño profesional y pulido que transmita seriedad. Los usuarios entregan datos financieros sensibles. |
| **Simplicidad** | 3 pasos: subir → analizar → reclamar. Cada pantalla tiene una sola acción principal. |
| **Accesibilidad** | Contraste WCAG AA, navegación por teclado, textos alternativos, etiquetas aria. |
| **Responsive** | Mobile-first donde aplica. Sidebar se convierte en drawer en móvil. |
| **Modo Oscuro** | Soporte completo con next-themes. Detección de preferencia del sistema. |

### Espaciado

- **Padding contenedor**: 1.5rem (24px) en desktop, 1rem (16px) en mobile
- **Gap entre tarjetas**: 1rem (16px)
- **Separación de secciones**: 2rem (32px)
- **Radio de borde**: 0.75rem (12px) para tarjetas, 0.5rem (8px) para inputs/botones
- **Sombras**: `shadow-sm` para tarjetas, `shadow-md` para modales

### Breakpoints (Tailwind)

| Breakpoint | Ancho | Cambios |

| **Default** (mobile) | < 640px | Sidebar → drawer (hamburguesa), stack vertical, tarjetas full-width |
| **sm** | ≥ 640px | Tarjetas en grid 1-2 columnas |
| **md** | ≥ 768px | Sidebar visible (colapsado a iconos), grid 2 columnas |
| **lg** | ≥ 1024px | Sidebar completo, grid 3-4 columnas |
